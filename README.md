
# 大连标普实验室样品全过程追溯系统
## BPLab Trace V4.4 入库任务接收预览版

本压缩包可直接解压后上传到 GitHub 仓库根目录，Streamlit 入口文件为 `app.py`。

### 本版新增

1. 客户信息基础资料管理，样品入库时从客户库选择。
2. 样品名称基础资料管理，样品入库时从样品名称库选择。
3. 样品编号规则改为 `BP年月日三位流水号`，例如 `BP20260722001`。
4. 样品编号允许手工修改；同一产品的独立样品可使用：
   - `BP20260722001-1`
   - `BP20260722001-2`
5. 同一次入库可设置多个独立样品，系统自动给出后缀编号，同时仍允许逐个修改。
6. 入库位置固定选择 `A区域` 或 `B区域`。
7. 样品状态固定选择 `完好` 或 `不完好`，并设置独立状态备注栏；不完好时备注必填。
8. 任务下发后，在实验人员账号显示提醒，并记录：
   - 任务下发时间
   - 提醒生成时间
   - 实验员接收时间
9. 实验员点击接收任务时确认：
   - 样品已收到，确认完好
   - 样品已收到，但存在异常
   - 尚未收到样品
10. “实验员接收任务”表示样品已经到手；时间轴同步记录接收时间和状态。
11. 错误入库样品在实验员接收前可以受控删除，必须填写原因；删除记录进入“已删除样品”，不从审计记录中消失。
12. 保留原V4.3的实验记录、复核、修改追踪、Word下载、样品归还和回库功能。

### 演示基础资料

系统首次运行会自动建立：
- 演示客户（待替换）
- 钴铬合金试样（演示）
- 氧化锆试样（演示）

正式客户清单和样品名称清单导入后，可在“基础资料”页面停用演示数据。

### GitHub / Streamlit部署

1. 解压压缩包。
2. 将解压后文件夹里面的所有内容上传到GitHub仓库根目录。
3. Streamlit入口选择 `app.py`。
4. 建议Python版本使用3.12。

### 说明

这是用于确认页面、流程和字段的预览版。收到正式客户信息库和样品名称清单后，可直接替换演示基础资料并生成正式更新包。
