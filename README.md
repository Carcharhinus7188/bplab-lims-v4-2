# BPLab Trace V5.0.1 完整落地版

大连标普实验室样品全过程追溯系统

## 已落地的业务主线

单位信息库 → 样品资料库 → 一份委托录入多个样品组 → 自动生成实体样品编号 → 任务包多选实验下发 → 实验员一次领用 → 每个实验独立原始记录 → 逐实验复核 → 整组样品一次归还 → 样品管理员回库 → 同一委托汇总一份检验报告 → 检测员、核验员、批准人分级签署。

## 样品编号

- 样品组基础编号：`BP年月日001`
- 实体样品编号：`BP年月日001-01`
- 数量为 6 时自动生成 `-01` 至 `-06`
- 即使数量为 1，也生成 `-01`

## 主要功能

- 一个委托编号可同时录入多个不同样品组
- 单位可标记为委托客户、生产单位、受委托生产企业
- 样品资料维护名称、规格型号、材料名称、单位、默认实验和默认方法
- 检测方法直接对应委托单勾选项
- 页面删除保密要求，委托单固定按“无要求”处理
- 收样员与样品管理员合并为“样品管理员”角色
- 一个样品组可将多个实验一次组成任务包下发
- 实验员一次领用、一次归还，不因单个实验完成而回库
- 每个实验仍生成独立原始记录并独立复核
- 任务编号、原始记录编号、复核编号一致
- 材料名称在下发任务时形成快照，实验员页面只读自动带出
- 11 个实验均有独立过程参数和原始数据字段
- 每个实验可预设设备、型号、编号、校准证书、有效期、软件和检测地点
- 支持上传电脑截图、实验过程照片、曲线、X 射线图像和原始数据文件
- 附件自动计算 SHA-256 校验值并形成索引
- 提供内部实验数据与附件追溯 Excel 模板下载
- 原始记录普通文字统一黑色；修改版仅修改值和修改说明使用红色
- 入库自动生成检验委托单和样品登记表
- 领用、归还自动生成样品领用归还登记表
- 同一委托全部实验完成后汇总生成一份半成品检验报告
- 报告按检测员确认、核验员核验、批准人批准发布
- 支持 PDF、PNG、JPG 电子签名上传
- 全部时间使用中国大陆时区 `Asia/Shanghai（UTC+8）`

## GitHub / Streamlit 部署

1. 将本文件夹中的全部内容上传到 GitHub 仓库根目录。
2. Streamlit 入口选择 `app.py`。
3. Python 建议 3.11 或 3.12。
4. 上传后执行 `Manage app -> Reboot app`。
