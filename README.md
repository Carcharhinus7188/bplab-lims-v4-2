# BPLab Trace V5.4 实验与设备动态配置版

本版整合了当前已确认的全部业务问题，并解决“实验和设备以后还会继续变化”这一核心要求。

## 1. 不再把实验—设备关系写死在程序里

88项最新版设备台账继续作为初始设备主数据，但实验与设备的关系改为数据库配置：

- 实验库独立维护；
- 设备库独立维护；
- SOP和原始记录模板独立维护；
- 通过“实验配置版本”关联方法、标准、地点、SOP、记录模板、软件和设备组合；
- 管理员可以增加实验、增加设备、停用设备、调整绑定和发布新版本，无须改代码。

## 2. 实验配置版本

每个实验可以建立 V1.0、V1.1、V2.0 等配置版本。只有“现行”版本用于新任务。

配置包含：

- 实验名称；
- 检测方法和检测依据；
- 专用或通用记录数据模板；
- 推荐检测地点；
- SOP版本；
- 原始记录模板版本；
- 软件名称/版本；
- 设备、标准器、夹具和辅助器具；
- 设备角色、是否必需、排序和说明。

## 3. 历史任务不会被后续配置覆盖

任务创建时保存完整配置快照和 SHA-256：

- 当时的实验名称、方法和依据；
- 配置版本；
- SOP和记录模板版本；
- 设备名称、管理编号、型号、量程、校准时间和状态；
- 设备在实验中的角色和是否必需。

以后更换设备、更新方法或发布新版本，不会修改已经建立的任务和原始记录。

## 4. 设备动态管理

设备库支持：

- 新增和修改设备；
- 启用、停用、维修、报废状态；
- 校准时间、责任人和状态说明；
- 当前设备台账导出；
- 现行实验配置设备矩阵导出；
- 审计追踪。

处于停用、维修或报废状态的设备不能作为新配置中的必需设备发布。

## 5. 实验动态管理

检测项目与方法库支持新增、修改、启用和停用。新增实验可以先使用“generic通用记录模板”，后续收到正式SOP和原始记录表后再建立专用数据模板。

系统仍不向用户显示实验内部标识，业务界面只显示“实验名称｜检测方法”。

## 6. 已保留的既有要求

- 删除金属—陶瓷结合三点弯曲实验；
- 保留金属—陶瓷结合裂纹萌生试验；
- 固定7个实验地点；
- 生产单位在委托层级统一选择；
- 样品资料库不设置生产单位；
- 每件实体样品采用 `BPYYYYMMDD001-01` 编号；
- 多实验采用任务包一次领用、一次归还；
- 每个实验独立原始记录和复核；
- 样品登记表采用 `DLBP-CX-P10-R01` 新模板；
- 88项最新版设备台账作为初始主数据；
- 原始记录保存设备使用确认、附件索引和配置快照；
- 同一委托最终汇总一份检验报告。
