# BPLab Trace V5.7.2 任务分配继承优化版

本版以 V5.7 全实验完整映射版为基础，保留原有数据库、编号规则、人员角色、菜单、Word母版和完整业务流程，只对任务下发、地点选择、实验时间与实时计算体验做小范围更新。

## V5.7.2 小更新

- 样品管理员分配任务包时只选择本次下发的“检测项目”；
- 不再把“检测项目＋检测方法”组合成可选项；
- 检测方法、检测依据、SOP和原始记录模板从前序委托及现行配置自动继承；
- 已选择项目的继承信息以只读表格展示，不能在任务分配阶段修改；
- 如需变更检测方法，必须返回前序受控配置流程处理并保留记录。

## V5.7.1 小更新

- 新建委托样品组和下发任务包时，检测项目不再自动勾选，必须按本次实际需求手动选择；
- 实验员接收任务包时，对每个实验子任务分别选择检测位置；同一任务包允许不同实验使用不同地点；
- 每个实验的地点保存到对应子任务和配置快照中，后续原始记录及报告使用实际选择地点；
- 实验开始、结束时间改为时间轴按钮记录，由系统自动写入北京时间并形成审计记录，不再手工输入；
- 原始测量值录入后，平均值、计算结果和符合性判定在当前样品卡片中实时刷新；
- 原有编号规则、人员操作窗口、六步实验记录页面和受控Word母版结构均未改变。

## 核心原则

- 实验员不查看Word模板原文、表格坐标或空白单元格；
- 委托、生产单位、样品、材料、编号、检测方法、地点和人员等前序数据自动带入，只读展示；
- 设备由任务配置快照自动带入，不要求实验员重新选择；
- 正常设备状态、样品状态和实验前检查设置为默认选项；
- 固定实验参数默认按现行SOP执行，只有选择“存在偏离”时才展开修改；
- 实验员只填写现场环境、设备/标准器核查、过程实测和原始测量值；
- 重复测量按样品卡片或测试面卡片录入，不使用Word表格式录入；
- 图片编号、曲线编号和原始文件索引由附件追溯自动关联；
- 计算结果由系统自动生成并只读展示；
- 提交检查依据真实业务必填项，而不是模板中的全部空白格；
- 提交后由后台把业务数据写入受控Word模板的原位置，不改变模板结构和版式。

## 实验员页面流程

1. 任务确认：前序数据只读，样品接收、编号和状态默认确认；
2. 设备与实验前检查：绑定设备逐台确认，正常为默认值；
3. 环境与参数：环境条件填写，固定参数默认按SOP执行；
4. 原始数据：按样品逐项填写，系统自动计算；
5. 异常与附件：正常完成为默认，异常时才展开说明；
6. 保存提交：按业务完整性检查后提交复核。

## 原始记录输出

程序继续使用用户提供ZIP中的10份受控原始记录表：

- R001 表面粗糙度；
- R004 金属—陶瓷结合裂纹萌生；
- R005 金属内部质量X射线灰度分析；
- R006 翘曲变形；
- R007 热膨胀系数；
- R009 陶瓷牙耐急冷急热；
- R010 弯曲性能；
- R011 维氏硬度；
- R012 色稳定性；
- R013 厚度测量。

模板母版不增加、删除、合并或拆分表格，也不追加设备表、附件表或修改附页。

10个实验已分别补充受控记录要求的环境前后值、设备核查、过程确认、逐件原始测量、复测/异常、自动计算和判定。厚度试验按母版记录每件试样3次重复、3个截面、每截面3个测点，共27个原始测量值。

当前业务字段清单共621项，详见：

- `business_field_mapping_summary.csv`
- `business_field_coverage_summary.json`

其中X射线像质计保留0.1 mm～1.0 mm各厚度点的3次原始灰度读数；耐急冷急热保留5个过程温度监测点；色稳定性保留开始、4个过程点和结束共6个运行监测点。

报告直接回填 `FORM_REPORT.docx` 母版，输出实际检验结果、标准要求、主要设备、环境范围、检验日期、样品说明和最终结论，禁止使用“详见原始记录”代替检验结果。

## 附件与Excel

附件追溯不设置软件或设备字段。附件与正式原始记录分离，通过委托编号、任务编号和样品编号关联。

单据中心保留：

`下载内部实验数据追溯Excel`

Excel包含实验总台账、附件索引、图片粘贴区、修改记录、复核记录和各实验追溯页，不替代正式原始记录表。

## 配置与历史追溯

- 实验、设备、SOP、记录模板和地点仍由管理员动态配置；
- 任务下发时冻结配置快照；
- 后续更换设备或更新实验配置，不改变历史原始记录；
- 修改版原始记录继续记录修改原因，并在模板中以红色标识变更内容。
- 全部实验复核通过并确认样品回库后自动生成待发布报告；首页和报告中心会幂等补建曾因中断遗漏的报告。
