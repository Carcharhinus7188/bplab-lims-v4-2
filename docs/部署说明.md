# Streamlit Community Cloud 部署

## GitHub

将本项目文件夹中的内容上传到仓库根目录。仓库首页应直接看到：

```text
app.py
requirements.txt
bplab/
templates/
docs/
```

不要上传：

```text
.venv/
__pycache__/
data/*.db
outputs/
attachments/
.streamlit/secrets.toml
```

这些内容已写入 `.gitignore`。

## Streamlit

在Streamlit Community Cloud新建应用：

```text
Repository：你的GitHub用户名/仓库名
Branch：main
Main file path：app.py
Python version：3.12
```

点击Deploy。

## 首次使用

系统会自动建立SQLite结构和初始角色账号。正式试运行前：

1. 修改初始密码；
2. 将应用设为私有并限制访问人员；
3. 在基础配置中补齐设备证书编号、溯源机构和有效期；
4. 补齐缺失的受控原始记录与报告母版；
5. 使用模拟数据完成全流程验证；
6. 不把Community Cloud本地文件作为正式记录的唯一存储。

## 更新

向GitHub提交新版本后，Streamlit会自动重新部署。数据库结构采用幂等初始化；但Community Cloud本地SQLite不保证持久化，正式运行应迁移到受控持久化数据库。
